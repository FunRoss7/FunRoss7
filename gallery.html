01
That’s a Rotate
Sean Parent · C++ Seasoning
A talk about something simple: loops. And something not simple at all: what those loops actually mean.
The idea is that most code is doing something that already has a name — find, count, transform, partition — and that writing it as a raw loop hides that meaning.
Once you see it, you start recognizing patterns everywhere. And once you recognize them, you stop writing certain kinds of code entirely.
It’s not about elegance. It’s about correctness and clarity stacking on top of each other over time.

02
What I took from it
Most of the code I’ve worked around wasn’t wrong — it just didn’t say what it was doing.
This talk gave me a way to look at that. Not to rewrite everything, but to ask a better question:

“What is this actually trying to do?”
Sometimes the answer is already in the standard library. Sometimes it isn’t. But either way, naming the operation changes how easy the system is to reason about.
It also changes how you break things apart. Functions get smaller. Boundaries get clearer. Bugs get easier to isolate.
03
Underneath it
Why it stuck
This isn’t really about loops.
It’s about recognizing structure early enough that you don’t build something harder than it needs to be.
A lot of engineering problems look complicated because we’re staring at them too close. Step back, and they collapse into something familiar.
When that happens, you get leverage.
